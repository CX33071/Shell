重定向符号

本质是改变命令的输入/输出来源/目的地

1.常用重定向符号有3种：

| 符号 | 名称                   | 作用                                                 |
| ---- | ---------------------- | ---------------------------------------------------- |
| >    | 标准输出重定向（覆盖） | 将命令的正常输出写入指定文件，覆盖文件原有内容       |
| >>   | 标准输出重定向（追加） | 将命令的正常输出写入指定文件，追加到文件末尾，不覆盖 |
| <    | 标准输入重定向         | 命令的输入不再来自终端，而是来自指定文件             |
| 2>   | 标准错误重定向         | 将命令的错误输出写入指定文件，覆盖i                  |

文件描述符是linux对输入输出对象的编号，0=标准输入（键盘）、1=标准输出（屏幕）、2=标准错误（屏幕），重定向的本质就是替换这些编号对应的对象，如把1换成文件

2.要实现重定向，需要解析重定向符号，修改子进程的输入/输出

具体要怎么实现呢？

解析命令行，分离命令参数和重定向符号+目标文件

在子进程中，关闭默认的标准输出fd1，打开目标文件并将其绑定到fd1

执行execvp时，命令就会把输出写到文件里
